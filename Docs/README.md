# مستندات پروژه MainProject

این پوشه شامل مستندات جامع و سازمان‌یافته پروژه است.

## ساختار مستندات

### فصل اول: معرفی برنامه و اهداف
- `01-معرفی.md`: معرفی کلی پروژه، اهداف، معماری و تکنولوژی

### فصل دوم: نمای کلی معماری
- `02-نمای-کلی-معماری.md`: بررسی سطح بالای شش بخش اصلی
  1. فرم‌ها (Forms)
  2. مدل‌ها (Models)
  3. منیجرها (Managers)
  4. لایه دسترسی به داده (Data Access Layer)
  5. پروسیجرها (Stored Procedures)
  6. جداول (Database Tables)

### فصل‌های تفصیلی (3-8)
- `03-فرم‌ها.md`: جزئیات کامل فرم‌ها و رابط کاربری
- `04-مدل‌ها.md`: مدل‌های داده و موجودیت‌ها
- `05-منیجرها.md`: لایه منطق کسب‌وکار
- `06-لایه-دسترسی-داده.md`: کلاس‌های DAL
- `07-پروسیجرها.md`: پروسیجرهای پایگاه داده
- `08-جداول.md`: ساختار پایگاه داده

### بخش‌های ویژه
- `سیم‌کشی/`: نقشه روابط بین لایه‌ها
  - `connections-map.md`: نقشه کامل اتصالات
- `عناصر-عمومی.md`: متدها، کلاس‌ها و پروسیجرهای مشترک

## استفاده برای AI

هر فایل شامل metadata tags برای تجزیه آسان توسط AI است:
- `#Form`: فرم‌ها
- `#Manager`: منیجرها
- `#DAL`: لایه دسترسی داده
- `#Procedure`: پروسیجرها
- `#Table`: جداول
- `#Connection`: روابط

## روش به‌روزرسانی

برای به‌روزرسانی مستندات:
1. از `@documentation-agent` استفاده کنید
2. Agent ابتدا تمام مستندات را می‌خواند
3. تغییرات را به صورت یکپارچه اعمال می‌کند
4. روابط و عناصر عمومی را به‌روز می‌کند

## یادداشت
- همیشه قبل از ویرایش، مستندات قبلی را بخوانید
- یکپارچگی را حفظ کنید
- روابط را ثبت کنید
- متدهای عمومی را دنبال کنید
